# Kindle Dashboard 使用说明

把一台越狱 Kindle PW3 当墙面显示器:WSL 端采集 CPU/GPU 数据,画成 1072×1448
灰度图,通过 SSH 推到 Kindle 用 `eips` 刷屏。**局部刷新 + sweep 历史图**,低损耗常驻。

- **`kindle_dash.py`** —— worker(采集 / 画图 / 推送循环)
- **`kindlectl`** —— 服务管理(开机自启 / 启停 / 状态)

---

## 0. 快速上手

```bash
cd /mnt/d/tmp/kindle
./kindlectl enable     # 装开机自启 + 立即启动 (无窗口常驻)
./kindlectl status     # 看运行 / 自启状态
./kindlectl stop       # 停 (会推告别屏)
./kindlectl disable    # 停 + 卸载开机自启
```

调试用(前台直接看输出):
```bash
python3 kindle_dash.py --interval 5            # 前台常驻, Ctrl-C 退出
python3 kindle_dash.py --once --save x.png     # 画一帧存本地看
python3 kindle_dash.py --message "你好 Kindle" # 推一屏文字
```

## 1. 界面

竖屏分上下两块,**上 CPU / 下 GPU**,结构相同:

- 顶部一行三列大数字 + 小标签:**使用率 / 温度 / 内存**(CPU 块内存=RAM,GPU 块=VRAM)
- 下方一大块 **btop 式上下镜像对称历史图**

**sweep(扫描)式历史图**:每次更新只在图右侧**新增一条竖线**,画满整宽后**清空、从左重画**。
好处是每 tick 只需局部刷新那一小条(+变化的数字),eink 损耗和延迟都最低。

开机/关机各有一屏中二**欢迎 / 告别**(`SYSTEM ONLINE / 高达驾驶员 Liuyi …`),
内容在 `kindle_dash.py` 顶部 `WELCOME_LINES` / `FAREWELL_LINES` 改。

---

## 2. 刷新策略(为什么不闪、为什么低损耗)

| 时机 | 做法 | 波形 | 耗时 |
|------|------|------|------|
| 每 tick | 只刷新「新增的竖条 + 变化的数字带」 | `du` 局部 | ~0.2–0.4s |
| 数字没变 | 跳过该数字带,不刷 | — | — |
| 每 `flush-every` 轮 | 整屏全刷洗掉 du 残影 | `gc16` | ~0.8s |
| 历史图画满 | 清空图表区从左重画 | `gc16` | — |
| 开机 / 关机 | 欢迎 / 告别整屏 | `gc16` | — |

> `du` 局部刷会累积轻微残影,靠周期性 `gc16` 整屏洗白。残影碍眼就调小 `--flush-every`。

## 3. 服务管理(kindlectl)

| 命令 | 作用 |
|------|------|
| `kindlectl enable` | 往用户启动文件夹放隐藏启动器 VBS(登录即起)+ 立即启动 |
| `kindlectl disable` | 停止 + 卸载开机自启 |
| `kindlectl start` | 立即后台启动(setsid 脱离终端,关终端不影响) |
| `kindlectl stop` | 优雅停止(SIGTERM → 推告别屏) |
| `kindlectl restart` | 重启 |
| `kindlectl status` | 运行中?开机自启已注册? |
| `kindlectl log` | `tail -f` 日志(`/tmp/kindle_dash.log`) |

**机制**:常驻的是 WSL 里的 python 循环,用 pidfile 单实例(重复启动自动退出)。
开机自启 = 用户**启动文件夹**里放隐藏启动器 `launch_kindle_dash.vbs`(登录时自动运行)
→ `wsl.exe` 后台跑 python。**无窗口、关终端不影响、不会被误关、免管理员**。
> (启动文件夹:`C:\Users\slyth\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup`)

注意:
- 触发点是**用户登录**,不是开机前。墙面机若自动登录则≈开机即起。
- 换机器改 `kindlectl` 顶部配置(`DISTRO`/`WSL_USER`/`SCRIPT`/`STARTUP`)。
- 关机告别屏是 **best-effort**(WSL 被杀宽限期短);手动 `stop` 则必现。

## 4. 前置条件(已配好,换机器才需重做)

### SSH 别名(`~/.ssh/config`,ControlMaster 复用连接)
```
Host kindle
    HostName 10.0.0.43
    User root
    ControlMaster auto
    ControlPath ~/.ssh/cm-%r@%h:%p
    ControlPersist 600
```
验证:`ssh kindle "echo ok; ls /usr/sbin/eips"`。

### Python 依赖
```bash
pip install --break-system-packages psutil pynvml pillow
```
`pillow`+`psutil` 必需;`pynvml` 读 GPU(读不到自动降级 `nvidia-smi`)。

### 字体
自动找,无需配置:**PingFang SC**(Windows 用户字体目录)→ 微软雅黑 →
DroidSansFallback → DejaVu。

### 整机读数来源(重要)
- **GPU 占用 / 温度 / 显存**:nvidia-smi/pynvml,硬件透传,读的是真显卡。
- **CPU 占用 / 温度 / 内存**:从 Windows 的 **LibreHardwareMonitor** 取(整机视角)。
  > ⚠️ 为什么不用 psutil:WSL 里 psutil 只能看到 **WSL 子系统**,游戏跑在 Windows
  > 侧时 CPU 会显示 ~0。所以 CPU 占用/温度/内存都走 LHM;LHM 挂了才降级到 psutil(不准)。
  - LHM 要求:
  1. LHM **管理员运行**常驻(OpenHardwareMonitor 太老,不支持 Zen5 9950X3D)。
  2. 开 LHM 的 **Remote Web Server**(端口 8085)。
  3. **防火墙放行 8085 入站**(已配)。
  - 默认自动用 **WSL 网关 IP 直连**(~7ms);没放行则 `--lhm-url http://localhost:8085/data.json`
    降级走 powershell(~0.4s)。已加 5s 缓存。LHM 没开则 CPU 温度显空,不影响其他。

## 5. 全部参数(kindle_dash.py)

| 参数 | 说明 | 默认 |
|------|------|------|
| `--interval N` | 循环间隔秒(`0`=不等待) | 5 |
| `--cpu-sample S` | psutil CPU 采样窗口秒(`0`=非阻塞,反映区间平均) | 1.0 |
| `--waveform WF` | 局部快刷波形(`du`/`gl16`/`a2` 等),越快残影越多 | `du` |
| `--flush-every N` | 每 N 轮 `gc16` 整屏洗白(`0`=从不;第 0 轮总整刷) | 10 |
| `--welcome-secs S` | 启动欢迎屏秒数(`0`=跳过) | 10 |
| `--no-farewell` | 退出不显示告别屏 | 关 |
| `--lhm-url URL` | LHM 地址(CPU 温度);默认网关 IP 直连,`localhost` 走 powershell | 自动 |
| `--temp-interval S` | CPU 温度缓存秒数 | 5.0 |
| `--once` | 只画一帧仪表盘(gc16 整刷)就退出 | 关 |
| `--message TEXT` | 推一屏文字(居中折行,支持中文/多行) | 无 |
| `--clear` | 刷新前先 `eips -c` 整屏洗白 | 关 |
| `--full` | 每轮都 `gc16` 整屏全刷(最干净最慢) | 关 |
| `--save PATH` | 把生成的 PNG 存本地(配 `--once`/`--message`) | 无 |
| `--host NAME` | SSH 别名 / 主机 | `kindle` |

> 服务(kindlectl)默认跑 `--interval 5 --cpu-sample 0`(改 `kindlectl` 里 `DASH_ARGS`)。

每 tick 日志示例:
```
CPU=12%/63°C RAM=3%  GPU=5%/52°C VRAM=13%
  耗时: 采集 0.01s | 画 0.00s | 刷 0.28s (2 列 + 2 数字块 [du]) | 总 0.29s
```

## 6. 故障排查

| 现象 | 排查 |
|------|------|
| 局部刷新 / push 失败 | `ssh kindle "ls /usr/sbin/eips"`;Kindle 同网段?休眠断 WiFi? |
| CPU 温度空 | LHM 没管理员运行 / web server 没开 / 防火墙没放行 8085 |
| GPU 显示 N/A | `nvidia-smi` 在 WSL 能否跑;不影响 CPU |
| 中文变 □ | 字体没找到 CJK;确认 PingFang/雅黑路径 |
| 残影明显 | 调小 `--flush-every` 或临时 `--full` |
| 启动报"已有实例" | 已在跑(pidfile `/tmp/kindle_dash.pid`);`kindlectl status` 查 |
| 开机没自起 | 是否已登录到 Windows;`kindlectl status` 看自启是否启用 |

## 7. 关键事实 / 相关文件

- 屏幕 **1072×1448** 竖向,8-bit 灰度;`eips` 全路径 `/usr/sbin/eips`(不在 PATH)。
- eips 局部刷新:`eips -g 图 -x X -y Y -w du`,坐标原点左上、单位像素(已验证)。
- 文件:`kindle_dash.py`(worker)、`kindlectl`(服务)、`launch_kindle_dash.vbs`
  (enable 时放进启动文件夹的隐藏启动器)、`kindle-dashboard-plan.md`(总体规划)。
